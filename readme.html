Readme: to update MIT HCI website for each PIs group

How to update hci.csail.mit.edu?


What content can I edit?

Each PIs group can edit the content that belongs to their group:
  • add or remove their own people including updating their photos, changing their personal webpage links etc.
  • add or remove their own research projects
  • add or remove their own classes

What content can I not edit?

For anything that is a global change, e.g. restructuring the website, updating things in the footer, adding new sections, please talk to Stefanie Mueller first and we can find a solution together.

Process for editing (thank you Rob!):


1. get the website code on your local: 

	git clone ssh://[CSAIL-USERNAME]@login.csail.mit.edu/afs/csail.mit.edu/proj/hci/git

2. Make your changes locally, then 

	git push

you should see a line of output 'remote: updating http://hci.mit.edu from git...'
which means that it's updating the website using what you just pushed.

(In the background, a simple hook script (/afs/csail.mit.edu/proj/hci/git/hooks/post-receive) updates the web folder (/afs/csail.mit.edu/proj/hci/www/data) whenever anybody pushes to the git repo.)

3. Adding new people with editing rights:
In order to have permission to access the git repo, people will need CSAIL accounts and to be members of the hci group.
Any member of the hci group can add somebody else using this command: 

	ssh [CSAIL-USERNAME]@login.csail.mit.edu pts adduser -group hci -user USERNAME


4. Checking/Querying users in a group:
In order to check the user's permission to access the git repo.
Any member of the hci group can query using this command: 

	ssh [CSAIL-USERNAME]@login.csail.mit.edu pts membership hci


5. Remove users in a group:
In order to remove the exists user permission to access the git repo.
Any member of the hci group can remove another user using this command: 

	ssh [CSAIL-USERNAME]@login.csail.mit.edu pts remove -group hci -user USERNAME
